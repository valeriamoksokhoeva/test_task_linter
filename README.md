# Linter Project
Линтер на Go проверяет логи по следующим правилам:
1. Сообщение должно начинаться со строчной буквы
2. Только на английском языке
3. Никаких спец. символов или эмодзи
4. Отсутствие чувствительных данных

Также в качестве бонусного задания выполнено:
- Автоматическое исправление ошибок SuggestedFixes 
- Возможность настраивать правила через конфигурационный файл `.linterlog.yaml` (можно включать/выключать правила)
- Можно указывать свои паттерны для проверки чувствительных данных (тоже в конфигурационном файле `.linterlog.yaml`)
- Реализован CI, запускающий тесты и сборку при каждом пуше/пуле


## Установка и сборка
### Технические требования
- Go 1.25.4 or later
### Способ 1 - ручной способ
1. Убедитесь, что установлена нужная версия Го.
2. Запустите команду сборки из корня проекта \
`go build -o mylinter ./cmd/`
3. Скопируйте исполняемый файл в свой проект \
`cp mylinter [путь к вашему проекту]`
4. Запустите линтер, так он проверит все файлы Go и файлы во вложенных папках \
`./mylinter ./...`
5. Запустите линтер с флагом -fix, чтобы исправить ошибки автоматически.
6. Запустите линтер с флагом -fix -diff, чтобы увидеть предложенные исправления.
7. Чтобы настраивать правила линтера и менять паттерны для чувствительных данных, создайте в своем проекте файл аналогичный файлу `.linterlog.yaml` с расширением .yaml. Настройте свои правила и запустите из своего проекта команду: 
`LINTERLOG_CONFIG="[ПОЛНЫЙ путь до вашего yaml файла]" ./mylinter ./...`
8. Чтобы запустить тесты:
`go test ./analyzer`
